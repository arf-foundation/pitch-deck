🧠 Bayesian risk scoring: expected‑loss minimisation

E[L] = P(Failure | Action) × Impact(cost) + β × Regret
  • P(Failure | Action) : posterior probability from ARF operational memory + causal effect estimator (conjugate priors, HMC, hyperpriors)
  • Impact : dollar exposure, SLO violation, regulatory fine
  • β × Regret : optional counterfactual regret term (what the agent would have lost had it acted)
  • Output: Approve · Deny · Escalate — deterministic, signed, auditable

❓ FAQ – “What if the Bayesian prior is wrong?”

  • Q: Doesn’t ARF add latency? A: Average decision latency < 50ms (core engine), <10ms for simple policies. Escalation path is asynchronous.
  • Q: How do you update the RAG memory without leaking sensitive data? A: We use local, encrypted vector stores with optional on‑prem deployment. No training data leaves your VPC.
  • Q: Can an agent game the risk score? A: The agent cannot see the full Bayesian model. Even if it reports artificially low confidence, ARF’s historical accuracy tracker adjusts the posterior.
  • Q: What about multi‑agent collusion? A: ARF’s policy algebra supports cross‑agent constraints (e.g., “only one agent may write to bucket X per minute”).
